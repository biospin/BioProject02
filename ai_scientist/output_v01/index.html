문헌 검색·정리 → 가설 설정 → 실험 수행 → 논문 작성 → 검수까지. 이 프로젝트는 그것을 완전 자율 기계가 아니라 "역할 분화 다중 에이전트 + 독립 Critic 게이트 + 사람 거버넌스"의 거버넌스 중심 하네스로 설계했고, 그 위에서 연구자 6명이 협업한다.

2-레이어 아키텍처 다중 에이전트 자동 검수 루프 사람 거버넌스 게이트

◆ 개요 — 설계의 4개 기둥 + 협업

한 줄 요약: 자동화의 목표는 사람을 빼는 것이 아니라, 사람이 판단해야 할 지점만 남기고 나머지 노동을 에이전트가 대신하는 것. 이 방향은 선행연구를 읽고 의도적으로 내린 포지셔닝이다.

설계상 중요한 두 구분

구분 1

계획 에이전트 ≠ 실행 입구

paper-orchestrator(agent)는 계획만 — 서브에이전트는 서브에이전트를 못 낳는다. 실제 실행은 paper-production-orchestrator(Skill)를 메인 루프(PI)가 돌린다.

구분 2

도메인 특수성 격리

H&E·병리·DepMap 지식은 spatialpatho-analyst 한 슬롯에 가둠. 문헌·집필·검수·발표 벤치는 어느 논문에도 옮겨 붙는다.

에이전트 발행 규칙: 서브에이전트가 무력해 보이는 건 권한이 아니라 프롬프트 문제. 첫 줄에 "슬래시/스킬 쓰지 말고 Bash로 Python 직접 실행" 명시, 코드를 copy-paste 형태로 직접 포함, env 경로(conda run -p /opt/envs/spatialpatho) 명시. 근거: CLAUDE.md 에이전트 발행·프롬프트 규칙
