# BIOP02 블로그 용어집 — 한 용어 = 한 표현

> `/blog` 스킬의 초심자 재작성이 따르는 프로젝트 용어집(규율은 `~/.claude/assets/blog/beginner-natural-writing.md` 3절).
> 시리즈 전편에서 **같은 전문용어는 같은 초심자 표현**으로 옮긴다. 새 글·재작성은 이 표를 기준으로 삼는다.
> 2026-07-09 BIOP02 9편을 주제별 6편으로 재구성하며 확정.

| 원어/전문용어 | 초심자용 표현 | 비유(선택) |
|---|---|---|
| WSI (whole-slide image) | 병리 슬라이드 사진 / 전체 슬라이드 영상. 첫 등장만 병기, 이후 "슬라이드" | 조직을 통째로 스캔한 초대형 사진 |
| 타일 / patch | 타일(조각) — 큰 슬라이드를 잘게 자른 작은 사각형 이미지 | |
| 임베딩 (embedding) | 임베딩 — 사진 한 장(또는 조각)을 고정 길이 숫자 벡터로 압축한 것. 이후 "임베딩" | 사진을 숫자 목록으로 요약 |
| 파운데이션 모델 (foundation model) | 파운데이션 모델 — 병리 이미지를 대량으로 미리 학습해 둔 대형 모델. 타일을 넣으면 특징을 압축한 벡터를 낸다 | |
| UNI / CONCH / EXAONE Path 2.0 | 이름은 유지하되 첫 등장 시 "무엇이고 어떻게 다른지" 한 줄. UNI=1024차원·타일 단위 / CONCH=512차원·타일 단위 / EXAONE Path 2.0=768차원·슬라이드 전체 입력 | |
| linear-probe (선형 분류기) | 임베딩은 그대로 두고 그 위에 단순한 분류기 하나만 얹어 성능을 보는 방법 — 임베딩이 얼마나 쓸 정보를 담았는지 가늠 | |
| MIL (multiple instance learning) / CLAM-SB | 다중 인스턴스 학습 — 슬라이드 하나를 수천 타일이 담긴 "가방"으로 보고, 어느 타일이 중요한지 모델이 스스로 가중치를 학습 | 가방 속 조각 고르기 |
| attention / attention map | 주목도 — 모델이 중요하게 본 타일. 이미지 위에 색으로 그리면 판단 근거를 눈으로 확인 | |
| MLP (다층 퍼셉트론) | 가장 단순한 신경망 기준선 — 슬라이드 평균 임베딩 한 벡터로 예측 | |
| site-confound / batch effect | 병원 배치효과 — 모델이 조직이 아니라 "어느 병원에서 스캔했는지"(색감·염색 습관)를 학습하는 정도. 이후 "배치효과" | 병원마다 다른 지문 |
| TSS site | 조직을 보낸 병원(사이트). TCGA에서 기관을 구분하는 코드 | |
| site-disjoint split | 병원이 겹치지 않게 학습/검증/시험을 나누기 | |
| AUROC | 판별 점수 — 1에 가까울수록 잘 맞힘, 0.5는 동전 던지기(무작위). 숫자는 그대로 표기 | |
| AUPRC | 소수 클래스까지 반영한 또 다른 판별 점수 | |
| external validation (외부검증) | 한 데이터(TCGA)로 학습한 모델을 다른 기관 데이터(CPTAC)에서 평가해 일반화를 확인 | 처음 보는 문제집 풀기 |
| Otsu 마스크 | 이미지 밝기로 배경(흰 유리)과 조직을 자동 구분하는 기법 | |
| coords (타일 좌표) | 슬라이드에서 타일을 뽑을 위치 목록 | |
| gated model | 승인을 받아야 내려받을 수 있는 모델 | |
| I/O starvation (입출력 굶주림) | GPU가 계산할 데이터가 제때 안 와서 노는 상태 | 재료가 안 와 노는 요리사 |
| SSD 스테이징 | 느린 HDD 원본을 빠른 SSD로 한 번 복사해 두고, 이후 잘게 흩어진 읽기를 전부 SSD에서 하는 것 | |
| ER / PR / HER2 | 면역조직화학(IHC)으로 본 유방암 수용체 상태(분자 표현형 라벨) | |
| PAM50 | 유전자 발현 기반 유방암 5개 아형 분류 | |
| hypothesis_only (가설 수준) | 확정 결론이 아니라 "가설"로만 다룬다는 표시 | |
| DRP (drug response prediction) | 약물 반응 예측 — 우리 프로젝트가 **아닌** 것(약물 구조를 입력으로 쓰지 않음) | |
| claim_level: sanity | 온전성 점검 — 파이프라인이 제대로 도는지 확인하는 수준, 정밀 결론 아님 | |

## 보존 주의 (이 프로젝트의 hedge — 절대 격상 금지)

- **임베딩 모델 비교의 claim_level은 sanity(온전성 점검)다.** "EXAONE가 더 나은 모델임을 증명했다"로 올리지 말 것. 예측 주장이 아니라 모델 선택용 신호.
- **EXAONE_slide_global의 우위(ER 0.9232 등)는 모델 자체 성능이 아니라 "슬라이드 집계 방식 + EXAONE 내부 타일링(Macenko@0.5mpp)"이 혼재된 것.** 동일 조건(mean-pool)인 EXAONE_patch_mean은 UNI와 대등(0.8517 vs 0.8551). "EXAONE가 UNI보다 좋은 모델"이라는 문장 금지 — deltas conflate model + tiling pipeline.
- **모델링 최종 Critic 판정은 caution이다(pass 아님).** HER2 외부검증 실패(ext_auc 0.53, auprc 0.12)는 완곡화하지 말 것 — 실제 실패다.
- **subtype_only 0.918은 상한선(ceiling)이지 바닥선(floor)이 아니다.** PAM50 아형을 입력으로 넣은 값이므로, 이미지 모델(CLAM 0.901)이 못 넘은 것은 한계이지 FAIL이 아니다. 이 구분을 뭉개지 말 것.
- **CPTAC 라벨은 공개 자료(cBioPortal brca_cptac_2020)에서 임시로 확보한 잠정치**다. 공식 라벨(jamie)이 오면 교체 예정. "확정 라벨"로 쓰지 말 것. 임시 라벨 기반 외부검증 결과는 재확인 필요(검토필요:).
- **배치효과는 제거가 아니라 통제**(site-disjoint 분할)했다. "배치효과를 제거했다/없앴다"로 쓰지 말 것 — UNI로 병원을 AUROC 0.998로 구분할 수 있다는 사실 자체는 남아 있다.
- **상관을 인과로 올리지 말 것.** site classifier가 병원을 잘 맞힌다 = 배치효과가 임베딩에 강하게 반영 = 관찰. "그래서 모델이 틀렸다"로 비약 금지.
- **작업자 귀속(attribution):** I/O 가속 = kkkim(BIOP02-38). OpenClaw JIRA-Slack 자동화 = braveji(BIOP02-85). 인프라 협업 정리 = kkkim. 모델링(CLAM/MLP) = sjpark, 그 교차리뷰 = kkkim. 데이터/manifest 초안 = jamie. 이 태그를 통합 과정에서 지우거나 kkkim에게 흡수시키지 말 것.
